http原理

http是一种无状态协议，不建立持久连接

    http完整请求过程的七个步骤

  • 建立TCP连接
  • 浏览器向服务器发送请求命令
  • 浏览器发送头信息
  • web服务器做出应答
  • web服务器发送应答头信息
  • web服务器向浏览器发送数据
  • web服务器关闭TCP连接

请求

请求方式get（幂等）用于信息获取 和post用于修改服务器资源

  • 请求由四部分组成
  • 1.请求方法或者动作（get / post）
  • 2.正在请求的URL
  • 3.请求头，客户端环境信息，身份验证等
  • 请求体（请求正文）

响应

一个响应有3部分组成 1.一个数字和文字组成的状态码，用来显示请求成功还是失败 2.响应头，响应头也和请求头一样包含许多有用信息，如服务器类型，日期时间，内容类型，长度等 3.响应体，也就是响应正文

状态码

  • 1xx:收到浏览器请求，正在进一步处理
  • 2xx:请求成功
  • 3xx:重定向，没有成功
  • 4xx：客户端提交请求有错误如404
  • 5xx:服务器错误
